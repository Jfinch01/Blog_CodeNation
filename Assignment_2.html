Describing copyright constraints on websites. 1. 5

A major constraint of the copyright law is that you can't just use any type of image or information you want. You would have to go through the actual author and creator of the content before you could use it unless you look for none copyrighted things in a refined google search.

If you choose to ignore all the copyright laws and use the information anyway then you could have legal action taken against you which can result in either a fine or jail time depending on what you're using the copyrighted item for. You're more likely to have legal action taken against you if you're looking to use their images or information for marketting and branding purposes without first gaining permission to do so from the author.

Appropriate file types for websites. 1. 7

  1. .AVI Audio/Video InterLeaved - This is a Video file type, supported by Windows. It's not very good for streaming videos sincde you have to download the entire file before you can view it. If it's left uncompressed then it can be a very large file type but by compressing it you might lose some video quality, although for the most part they do have very high video quality going up to 1080p.
  2. .CCS Cascading StyleSheet - This basically covers how the website is going to look, using different style types and can be created in any sort of text editor such as NopePad or anything along those lines.
  3. .EXE Executable file - When you click on this file type, either in browser or if it gets sent to you, it runs a file automatically and doesn't always require require admin rights in order to run depending on the code of the .EXE file.
  4. .HTML HyperText Markup Language - This is what most website pages load in, it's basically the skeleton for any site, setting out all of the data before you apply CSS to style it.
